THE PROBLEM:
  AutoCAD script files need proper line breaks to simulate user input.
  When WBLOCK runs interactively, you press ENTER after each prompt.
  The script must include blank lines to simulate those ENTER keys.

OLD SCRIPT (BROKEN):
  -WBLOCK "C:\path\block.dwg" = BlockName
  
  This tries to put everything on one line.
  AutoCAD doesn't know where one parameter ends and another begins.
  Result: Command ignored, nothing happens.

NEW SCRIPT (FIXED):
  -WBLOCK              â† Command name
  "C:\path\block.dwg"  â† File path
                       â† ENTER (blank line)
  =BlockName           â† Block name
                       â† ENTER (blank line)
  
  AutoCAD reads line-by-line, treating blank lines as ENTER.
  Result: WBLOCK executes correctly, DWG file created.
